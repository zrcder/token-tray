# 写代码写一半突然 429？这个菜单栏小工具帮你实时盯住大模型额度

> TokenTray：一个 5MB 的 macOS 菜单栏工具，把智谱 GLM / OpenAI / Claude 的 API 用量放到你的眼皮底下。

---

## 你一定遇到过这个场景

正在用 Cursor、Claude Code 或者自己写的脚本猛跑 AI 补全，代码写得飞起 —— 然后突然：

```
Error: 429 Too Many Requests
您的账户已达到使用上限
```

心态直接炸了。

更烦的是，你想知道**还剩多少额度**、**什么时候重置**，得：
1. 打开浏览器
2. 登录开放平台控制台
3. 翻到用量页面
4. 看一堆数字

等你切回编辑器，思路早就断了。

## TokenTray：瞄一眼就知道

我做了一个 macOS 菜单栏小工具叫 **TokenTray**，把大模型的额度信息放到菜单栏右上角，**一秒可见**。

安装后，你的菜单栏会多出一个这样的图标：

```
… 🔋 🔊 📶 📅  智 22%
                ↑ 这就是 TokenTray
```

`22%` 表示你当前所有额度窗口里**最高的那个使用率**。扫一眼就知道安不安全。

**点开它**，展开完整的用量面板：

```
🟢 智谱 GLM · MAX

   5 小时 Token
   ██░░░░░░░░  22%
   ⏳ 3h 12m 后重置

   每周 Token
   █░░░░░░░░░  5%
   ⏳ 5d 8h 后重置

   MCP 月度
   ░░░░░░░░░░  0%  (0 / 4.0K)

   更新于 14:32:08
```

三个关键信息一目了然：
- **5 小时窗口**：智谱 Coding Plan 的短窗口，最容易撞墙
- **每周窗口**：7 天长窗口，撞了要等更久
- **MCP 月度**：搜索/网页阅读等工具调用配额

颜色从绿变黄变红，不用看数字就知道状态。

## 它不只是一个查额度工具

### 1. 状态灯，扫一眼就走

- 🟢 **绿色**：用量 < 70%，随便用
- 🟡 **黄色**：70% - 90%，悠着点
- 🔴 **红色**：> 90%，该收手了

不用读数字，看颜色就行。

### 2. 重置倒计时

每个额度窗口都显示**距离重置还有多久**。

```
⏳ 3h 12m 后重置    ← 5 小时窗口，下午就知道能不能续上了
⏳ 5d 8h 后重置     ← 每周窗口，心里有数
```

撞限流的时候看一眼倒计时，决定是去喝杯咖啡还是切别的活儿干。

### 3. 应用内配置，不碰文件

设置 API Key 不用编辑配置文件、不用搞环境变量 —— 点菜单里的 **「⚙ 设置…」**，弹出原生 macOS 对话框，粘贴 Key，点保存，立刻生效。

换 Key、删 Key 都在菜单里完成。

### 4. 多模型架构，为扩展而生

TokenTray 内部用 Provider 抽象设计：

```
当前：智谱 GLM ✅
规划中：OpenAI / Anthropic / DeepSeek 🚧
```

每个模型供应商实现同一个接口，加一个新供应商只需写一个文件。未来你可以在一个菜单里同时看到 GLM + GPT + Claude 的额度。

## 技术选型：为什么是 Go

TokenTray 用 **Go + AppKit CGO** 构建，不是 Electron、不是 Python、不是 Swift。

| 方案 | 体积 | 启动 | 体验 |
|------|------|------|------|
| Electron | 150MB+ | 慢 | Web 套壳，割裂感 |
| Python + rumps | 24MB | 慢 | PyInstaller 打包坑多 |
| Swift | 2MB | 快 | 最 native，但依赖 Xcode |
| **Go + CGO** | **5MB** | **快** | **原生 AppKit，零运行时依赖** |

Go 的优势：
- **5MB 单文件**，不拖运行时，不占内存
- **CGO 直连 AppKit**，菜单栏渲染跟原生 App 一样
- **交叉编译友好**，构建脚本一行命令
- **Provider 接口清晰**，加新模型极快

## 一些踩坑记录

做这个小工具的过程中踩了不少 macOS 开发的坑，分享几个：

### 1. 智谱的认证头没有 Bearer

大部分 API 的认证头长这样：`Authorization: Bearer sk-xxx`

智谱的 Monitor 接口长这样：`Authorization: xxxxxx.yyyyyy` —— **直接裸 Key，没有 Bearer 前缀**。

这个坑如果没发现，401 到怀疑人生。

### 2. Coding Plan 有两个 Token 窗口

智谱 Coding Plan（包月套餐）返回**两个** `TOKENS_LIMIT` 条目：
- 5 小时滚动窗口
- 7 天滚动窗口

按 `nextResetTime` 排序就能区分。按量付费账号则没有这种结构。

### 3. macOS 菜单栏 App 的线程陷阱

NSStatusBar 的 status item **必须在主线程创建和操作**。Go 的 goroutine 模型和 macOS 的主线程要求天然冲突，用错线程会导致 status item 静默失败 —— 进程在跑，注册了，但图标死活不出现。

这个 bug 花了我大半天排查。

### 4. macOS 15.3 的 WindowServer 渲染 bug

在 macOS 15.3 上，即使代码完全正确，新创建的菜单栏图标也可能不渲染。升级到 15.7 后修复。如果你做菜单栏工具遇到类似问题，先查 macOS 版本。

## 写在最后

做 AI 开发，API 额度管理是一个很基础但被忽视的需求。大模型厂商的控制台做得越来越花哨，但你真正需要的信息 —— **还剩多少、什么时候重置** —— 反而要点好几层才能看到。

TokenTray 的理念很简单：**把最重要的信息放到最显眼的地方**。

菜单栏是 macOS 上最容易被瞥到的位置，适合放那种「不需要交互、但需要随时知道」的信息。电量、时间、网络状态都是这么做的。API 额度也应该在这里。

如果你也在用智谱 GLM 或其他大模型 API，试试 TokenTray，让额度焦虑少一点。
